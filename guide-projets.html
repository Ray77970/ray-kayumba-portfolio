Wie Projekte funktionieren

Jedes Projekt besteht aus zwei Teilen: einer Karte in der Projektübersicht (works.html) und einer Detailseite im Ordner /projects/.

Jedes Bild im images[]-Array eines Projekts erzeugt eine eigene Karte in der Übersicht — alle verlinken auf dieselbe Detailseite.
01
Projekt hinzufügen

Öffne works.html und suche das PROJECTS-Array im <script>-Block (ca. Zeile 80). Füge ein neues Objekt hinzu:

works.html — PROJECTS[]
// Neues Objekt in das Array einfügen:
{
  title: 'Mein Projektname',
  tags: ['Tag1', 'Tag2', 'Tag3'],
  category: 'AI',  // ← siehe Tabelle unten
  link: 'projects/mein-neues-projekt.html',
  images: [
    'Images/foto_01.jpg',
    'Images/foto_02.jpg',
    'Images/foto_03.jpg',
  ]
}

Verfügbare Kategorien:

category-WertFilterbuttonVerwendung
AIKIKI-Projekte, Automationen, Tools
UI/UXUI/UXApps, Websites, Dashboards
ProductionProduktionVisuelle Produktion, AI-Bilder
IdentityIdentitätBranding, Logos, Systeme
02
Detailseite erstellen

Kopiere eine bestehende Datei aus /projects/ und benenne sie nach deinem Slug um (z. B. mein-neues-projekt.html).

Dann passe folgende Stellen an:

1 · Seitentitel (im <head>)
<title>Projektname — Ray Kayumba</title>
2 · Hauptbild (linke Seite)
<img class="lb-main-img" id="mainImg"
     src="../Images/foto_01.jpg"
     alt="Projektname">
3 · Projekttitel & Beschreibung (rechte Seite)
<div class="lb-proj-title">Projektname</div>
<div class="lb-proj-desc">Beschreibung auf Deutsch...</div>
4 · Tags
<span class="ptag">Tag1</span>
<span class="ptag">Tag2</span>
<span class="ptag">Tag3</span>
5 · Metadaten (aufklappbarer Bereich)
<!-- Rolle -->
<span class="lb-detail-val">Brand Designer</span>

<!-- Art -->
<span class="lb-detail-val">Markenidentität</span>

<!-- Stack -->
<span class="lb-detail-val">Figma · Adobe CC</span>

<!-- 4. Feld (frei wählbar: Ergebnis, Bereich, etc.) -->
<span class="lb-detail-val">100+ Assets</span>
6 · Bilder-Array für den Slideshow (im <script> unten)
const IMAGES = [
  '../Images/foto_01.jpg',
  '../Images/foto_02.jpg',
  '../Images/foto_03.jpg',
  // beliebig viele hinzufügen
];
03
Kategorie ändern

Öffne works.html, suche das Projekt-Objekt im PROJECTS-Array und ändere nur den category-Wert:

// Vorher:
category: 'Identity',

// Nachher:
category: 'UI/UX',
→
Der Filterbutton und die Sichtbarkeit in der Übersicht werden automatisch aktualisiert — keine weiteren Änderungen nötig.
04
Bilder hinzufügen

Bilder gehören immer in den Ordner Images/ im Root des Projekts.

Referenz inPfad-Syntax
works.html'Images/dateiname.jpg'
projects/*.html'../Images/dateiname.jpg'
→
JPG empfohlen. Für Web idealerweise unter 400 KB pro Bild komprimieren (z. B. mit Squoosh oder TinyPNG).
05
Zusammenfassung in 4 Schritten
1
Bilder vorbereiten JPG-Dateien in den Ordner Images/ legen
2
works.html öffnen Neues Objekt in PROJECTS[] mit title, tags, category, link, images einfügen
3
Detailseite duplizieren Bestehende .html aus /projects/ kopieren und umbenennen
4
Inhalte anpassen Titel, Beschreibung, Tags, Metadaten und IMAGES[]-Array bearbeiten
